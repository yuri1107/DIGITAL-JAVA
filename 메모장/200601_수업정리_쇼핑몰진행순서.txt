회원추가 -> 구매 (등록) -> 구매리스트(에저장)

회원추가(판매자) -> 관리자가 승인(멤버테이블에 authority속성을 판매자로 변경) 
	-> 제품(을추가) -> 제품등록(을 추가) -> 제품등록리스트(에추가)

회원이 제품을 구매했을때 DB에서 일어나는 상황

1. 회원의 아이디가 구매의 구매자와 같아짐.
2. 제품이 구매리스트에 담김.
3. 구매리스트의 해당 제품 구매수량만큼 제품의 재고(제품등록리스트)가 마이너스.
4. 구매에서 구매번호가 생기고 총가격이 (구매리스트의 구매수량*제품의 가격) 만큼 누적됨.
**위에는 내생각 아래는 쌤말씀

1. 구매(purchase) 테이블 insert 이벤트
2. 구매리스트(purchase_list)테이블 insert 이벤트
	=> 구매한 제품 종류 만큼
3. 제품등록리스트(resitration_list) 테이블 update 이벤트
	=> 재고량
4. 구매(purchase) 테이블 update이벤트 
	=> 총가격

**3번과 4번은 서로 독립적이기 때문에 순서가 바뀌어도 상관이 없음
***3,4번은 순서가 바뀌어도 된다. 각 정보들이 서로 영향을 받지 않기 때문에.

관리자가 판매글을 등록했을때 DB에서 일어나는 상황
(제품등록리스트가 업데이트될때)

상황1 - 판매글을 등록하기 위한 제품이 이미 있는 경우
1. 제품등록 (product_registration)테이블 insert이벤트
2. 제품등록리스트(registration_list)테이블 insert이벤트
	=>등록하려는 제품 종류만큼

상황2 - 판매글을 등록하기 위한 제품이 없는 경우
1. 제품(product)테이블 insert이벤트 => 등록하려는 제품 종류 만큼
2. 제품등록 (product_registration)테이블 insert이벤트
3. 제품등록리스트(registration_list)테이블 insert이벤트
	=>등록하려는 제품 종류만큼


비회원이 판매자로 등록되기 위한 과정
1. member 테이블에 insert 이벤트
	= > 회원가입
2. member 테이블에 update 이벤트
	= > 관리자가 해당 회원의 권한(m_authority)를 변경
